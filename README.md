# КиївЧас — як запустити не одним файлом

У тебе зараз два режими роботи цього проєкту:

1. **Просто відкрити `kyiv-chas.html`** — працює одразу, ніде нічого не
   треба ставити, але карта показує *симуляцію* (плями/радіуси), не
   справжні вулиці.
2. **Запустити повноцінний бекенд** (`kyiv_backend.py`) — тоді та сама
   сторінка почне показувати РЕАЛЬНИЙ полігон, порахований по графу вулиць
   Києва й реальному розкладу транспорту. Це і є "точність до вулиці",
   наскільки вона взагалі досяжна без платних живих даних про затори.

Різниця на карті: у режимі 1 клік по карті малює кольорову пляму. У
режимі 2 — той самий клік малює нерівний контур, що повторює форму
вуличної мережі (де немає дороги — туди й полігон не залазить).

---

## Що тобі знадобиться

- Python 3.10+ встановлений на комп'ютері
- Стабільний інтернет (перший запуск тягне граф вулиць Києва — кілька
  сотень МБ, займає 5-15 хв залежно від швидкості з'єднання)
- ~15 хвилин вільного часу на перший запуск

## Крок 1 — встанови залежності

Відкрий термінал (Термінал на macOS/Linux, PowerShell чи cmd на Windows)
у папці з файлами проєкту:

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Крок 2 — завантаж реальні дані Київпастрансу (GTFS)

Я НЕ зміг автоматично витягнути точне посилання на завантаження з цієї
сесії (портал data.kyivcity.gov.ua — це JS-застосунок, мій інструмент
фетчу бачить лише заголовок сторінки, не тіло з реальними ID ресурсів).
Зроби це вручну, це 2 хвилини:

1. Відкрий https://data.kyivcity.gov.ua у браузері
2. Пошук → "Розклад руху міського електричного та автомобільного транспорту"
3. Знайди ресурс з назвою на кшталт **GTFSStatic** (тип: ZIP чи TXT-архів)
4. Натисни завантажити, збережи файл як `kyivpastrans_gtfs.zip` **у ту ж
   папку**, де лежить `kyiv_backend.py`
5. Якщо на порталі немає прямого GTFS — запасний варіант:
   https://data.gov.ua → пошук "Київпастранс GTFS" (загальнодержавний
   дублікат порталу міста, іноді має те, чого нема на міському)

Якщо взагалі не знайдеш — скажи мені в наступному повідомленні "не можу
знайти GTFS", і я підкажу ще 2-3 запасних джерела для пошуку.

## Крок 3 — запусти бекенд

```bash
uvicorn kyiv_backend:app --reload
```

Перший запуск довгий (тягне граф вулиць з OpenStreetMap і кешує його у
`kyiv_street_graph_walk.pkl` — наступні запуски вже за секунди, бо
`get_or_build_street_graph()` бере граф з диска замість повторного
завантаження). Побачиш у консолі:

```
[build] тягну граф вулиць з OpenStreetMap (walk) — це триватиме кілька хвилин...
[ready] сервер готовий приймати запити на /isochrone
```

Перевір, що працює, відкривши в браузері:
```
http://localhost:8000/isochrone?lat=50.45&lng=30.52&minutes=45
```
Має повернутись GeoJSON (текст, що починається на `{"type": "Feature"...`).

## Крок 4 — відкрий сайт і підключи бекенд

1. Відкрий `kyiv-chas.html` як завжди (подвійний клік)
2. Клікни точку на карті
3. У панелі зліва знайди кнопку **"🔌 Підключити точний бекенд"** і натисни
4. Якщо все запущено правильно — з'явиться напис "✅ Показано справжній
   полігон..." і на карті намалюється контур по реальних вулицях (золотий
   контур поверх симуляції)

Якщо бачиш "⚠️ Не вдалось з'єднатись" — перевір, що `uvicorn` досі
запущений у терміналі (крок 3) і що адреса саме `localhost:8000`.

## Структура проєкту

```
kyiv-chas.html              — фронтенд (те, що бачить користувач)
kyiv_backend.py             — бекенд: граф вулиць + GTFS + Дейкстра + API
kyivpastrans_gtfs.zip       — реальні дані Київпастрансу (завантажуєш сам, крок 2)
requirements.txt            — python-залежності
kyiv_street_graph_walk.pkl  — кеш графа вулиць (створюється автоматично після 1-го запуску)
kyiv_cache.sqlite           — власна база: кешовані ізохрони, щоб повторні запити були миттєвими
```

## Чому саме так, а не "все в одному файлі"

- **Граф вулиць Києва** — це десятки тисяч вузлів і сотні тисяч ребер.
  Браузер може намалювати картинку такого розміру, але не може порахувати
  по ньому Дейкстру за прийнятний час і не повинен зберігати стільки
  даних локально при кожному відкритті сторінки.
- **GTFS-розклад** — це живі, реальні дані перевізника, які регулярно
  оновлюються. Вшивати їх намертво в HTML-файл означає, що вони одразу
  застаріють.
- Тому: **фронтенд (HTML) — це те, що людина бачить і клікає. Бекенд
  (Python-сервер) — це те, що рахує.** Так влаштовані всі мапи доступності
  у світі (Google Maps, TravelTime, Mapnificent) — жодна з них не рахує
  маршрути прямо в браузері користувача.

## Якщо щось не запускається

Найчастіші проблеми:
- `ModuleNotFoundError` → не активований venv або не виконався `pip install -r requirements.txt`
- Зависає на "тягну граф вулиць" дуже довго (>30 хв) → OpenStreetMap
  Overpass API іноді перевантажений, спробуй пізніше або зменш зону
  (заміни `PLACE = "Kyiv, Ukraine"` на менший район, наприклад
  `"Solomianskyi District, Kyiv, Ukraine"`, для першого тесту)
- `/isochrone` повертає порожній результат → перевір, що
  `kyivpastrans_gtfs.zip` дійсно лежить поруч і не пошкоджений
  (спробуй розпакувати вручну — має бути 6-9 файлів `.txt` всередині)

## Наступний рівень (якщо захочеш іти далі)

Найкращий спосіб продовжити саме цю частину роботи — **Claude Code**: він
працює з відкритим доступом до інтернету на твоєму комп'ютері і зможе
сам завантажити GTFS, підключити граф, налагодити помилки наживо й навіть
задеплоїти сервер, а не просто написати код "наосліп", як я зараз.
